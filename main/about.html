Câu 1 [cite: 2]

1. Position: relative [cite: 3]

  • Phần tử vẫn nằm trong luồng (normal flow) của trang. [cite: 4]
  • Có thể dịch chuyển bằng top, left, right, bottom. [cite: 5]
  • Khoảng trống ban đầu của phần tử vẫn được giữ. [cite: 6]

Ví dụ: [cite: 7]

.box {
    position: relative; [cite: 8, 9]
    left: 20px; [cite: 10]
    top: 10px; [cite: 11]
} [cite: 12]

Ứng dụng: [cite: 13]

  • Dịch chuyển nhẹ icon. [cite: 14]
  • Làm phần tử cha để chứa phần tử absolute. [cite: 15]
  • Điều chỉnh vị trí mà không làm hỏng layout. [cite: 16]

2. Position: absolute [cite: 17]

  • Phần tử bị loại khỏi luồng của trang. [cite: 18]
  • Định vị theo phần tử cha gần nhất có position khác static. [cite: 19]
  • Nếu không có cha thì căn theo body. [cite: 20]

Ví dụ: [cite: 21]

.parent {
    position: relative; [cite: 22, 23]
} [cite: 24]
.child {
    position: absolute; [cite: 25, 26]
    top: 10px; [cite: 27]
    right: 10px; [cite: 28]
} [cite: 29]

Ứng dụng: [cite: 30]

  • Nút đóng (X) [cite: 31]
  • Badge thông báo [cite: 32]
  • Menu dropdown [cite: 33]
  • Popup, Tooltip [cite: 34, 35]

3. Responsive Layout [cite: 36]

Ví dụ: Khi thay đổi kích thước màn hình, badge vẫn bám đúng góc của card. [cite: 37, 46]

.card { position: relative; } [cite: 38, 39, 40]
.badge { position: absolute; top: 5px; right: 5px; } [cite: 41, 42, 43, 44, 45]

Ảnh hưởng khi deploy website: [cite: 47]

  • relative giúp layout ổn định trên nhiều màn hình. [cite: 48]
  • absolute nếu dùng sai dễ bị lệch trên điện thoại. [cite: 49]
  • Khi deploy cần kết hợp: Flexbox, Grid, Media Query, Đơn vị (%, rem, vw) để website tương thích nhiều thiết bị. [cite: 50, 51, 52, 53, 54, 55]

Câu 2: Quy trình đăng ký Domain và Hosting [cite: 56, 57]

  1. Bước 1: Đăng ký tên miền (Ví dụ: Abc.com) [cite: 58, 59, 60, 61]
  2. Bước 2: Thuê Hosting. Có thể dùng: Netlify, GitHub Pages, Vercel [cite: 62, 63, 64, 65]
  3. Bước 3: Upload source code (Index.html, Style.css, Script.js) [cite: 66, 67, 68]
  4. Bước 4: Cấu hình DNS [cite: 69, 70]

Nhiệm vụ của DNS:

  • Chuyển tên miền thành địa chỉ IP của hosting. Giúp trình duyệt tìm đúng máy chủ. [cite: 71, 72, 73]
  • Ví dụ: Abc.com -> DNS -> 104.xxx.xxx.xxx [cite: 74, 75]

Hai công cụ deploy miễn phí: [cite: 76]

  • GitHub Pages: Ưu điểm là miễn phí, kết nối GitHub, dễ dùng. [cite: 77, 78, 79]
  • Netlify: Ưu điểm là deploy tự động, có HTTPS, hỗ trợ Domain riêng, Continuous Deployment. [cite: 80, 81, 82, 83, 84, 85]

Rủi ro bảo mật: [cite: 86]

Nếu cấu hình sai có thể gặp các vấn đề: [cite: 87]

  • Lộ mã nguồn [cite: 88]
  • Sai DNS khiến website không truy cập được [cite: 89]
  • Không bật HTTPS gây mất an toàn dữ liệu [cite: 90]
  • Lộ API Key [cite: 91]
  • Bị tấn công nếu quyền truy cập quá rộng [cite: 92]

Câu 3: Thuộc tính Let, Var [cite: 93]

Thuộc tính Let Var
Scope Block Function
Khai báo lại Không Có
Gán lại Có Có
Hoisting Có nhưng TDZ (Temporal Dead Zone) Có (undefined)

(Dữ liệu bảng dựa trên tài liệu cung cấp) [cite: 94]

Ví dụ: [cite: 95]

  • var a = 10; var a = 20; (Hợp lệ) [cite: 96]
  • let b = 10; let b = 20; (Lỗi) [cite: 97]
  • const PI = 3.14; PI = 5; (Lỗi) [cite: 98]

Ví dụ lỗi thường gặp (TDZ): [cite: 99]

console.log(x); [cite: 100]
let x = 5; [cite: 101]
// Kết quả: ReferenceError [cite: 102, 103]

Câu 4: Event Delegation là gì? [cite: 117, 118]

Là kỹ thuật chỉ gắn một Event Listener lên phần tử cha, sau đó xử lý sự kiện cho các phần tử con thông qua cơ chế Event Bubbling. [cite: 119]

Ví dụ: [cite: 120]

HTML: [cite: 121]

<ul id="list"> [cite: 122]
    <li>A</li> [cite: 123]
    <li>B</li> [cite: 124]
    <li>C</li> [cite: 125]
</ul> [cite: 126]

JavaScript: [cite: 127]

document.getElementById("list").addEventListener("click", function(e) { [cite: 128]
    if(e.target.tagName === "LI") { [cite: 129]
        alert(e.target.innerText); [cite: 130]
    } [cite: 131]
}); [cite: 132]

Nếu sau này thêm phần tử động bằng JS: [cite: 133]

let li = document.createElement("li"); [cite: 134]
li.innerText = "D"; [cite: 135]
document.getElementById("list").appendChild(li); [cite: 136]

...thì phần tử mới vẫn hoạt động mà không cần gắn thêm Event Listener. [cite: 137]
